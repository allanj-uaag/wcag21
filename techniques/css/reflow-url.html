Allowing for Reflow with Long URLs

Metadata

css

sufficient

When to Use

This technique is applicable to Cascading Style Sheet / HTML technologies.

Description

Long URLs can break reflow when the page is zoomed. The objective of this technique is to present URLs without introducing a horizontal scroll bar at a width equivalent to 320 CSS pixels or a vertical scroll bar at a height equivalent to 256 CSS pixels. This is done by using CSS techniques that adapt to the available viewport space.

Examples

Example 1: Breaking long URLs

Using the follwoing CSS will cause long URLs to break at appropriate places (hyphens, spaces, etc.) without causeing reflow.

List of CSS declaration used and why they are used:

  • overflow-wrap: break-word; and word-wrap: break-word;: Allows long URLs to be broken, wrapped, and not overflow the available space. They are synonamous, however due to variable browser implementation, use both.
  • word-break: break-all: Allows words to be broken and wrapped between letters.
  • hyphens: auto: Allows words to be hyphenated and wrapped
    • a { overflow-wrap: break-word; word-wrap: break-word; word-break: break-all; hyphens: auto; } /* May need to add browser specific rule for hyphens to work. Not currently supported in Blink -ms-hyphens: auto; -moz-hyphens: auto; -webkit-hyphens: auto;*/
